# Internationalization (i18n)

Ultra-lean multi-language documentation support with URL-based routing, RTL auto-detection, and locale fallback chains.

## Overview

The i18n system provides essential internationalization features without bloat. Content organization follows a simple directory structure with automatic locale detection and SEO optimization.

**Philosophy:** Ship only what's necessary. No translation UIs, no browser detection, no progress tracking—just clean, URL-based multi-language routing.

## Features

- **URL-Based Routing** - `/docs/guide` (English) → `/es/docs/guide` (Spanish)
- **Fallback Chains** - `es-MX` → `es` → `en` (default)
- **RTL Auto-Detection** - Automatically detects right-to-left languages (Arabic, Hebrew, Persian, Urdu)
- **Language Switcher** - Dropdown component for locale selection
- **SEO Optimization** - Automatic hreflang tags for search engines
- **Locale-Aware Search** - Separate search indexes per language

## Quick Start

### 1. Organize Content by Locale

```
docs/
  en/                 # English (default)
    getting-started.md
    api-reference.md
  es/                 # Spanish
    getting-started.md
    api-reference.md
  zh/                 # Chinese
    getting-started.md
    api-reference.md
```

**URL Structure:**
- English: `/docs/getting-started` (no locale prefix)
- Spanish: `/es/docs/getting-started`
- Chinese: `/zh/docs/getting-started`

### 2. Configure i18n

```typescript
// src/lib/config.ts
import type { I18nConfig } from '@goobits/docs-engine/utils';

export const i18n: I18nConfig = {
  defaultLocale: 'en',
  locales: [
    { code: 'en', label: 'English' },
    { code: 'es', label: 'Español' },
    { code: 'zh', label: '中文' },
    { code: 'ar', label: 'العربية', dir: 'rtl' }  // RTL example
  ]
};
```

### 3. Add Language Switcher

```svelte
<script lang="ts">
  import { LanguageSwitcher } from '@goobits/docs-engine/components';
  import { i18n } from '$lib/config';
  import { page } from '$app/stores';

  const currentLocale = detectLocaleFromPath($page.url.pathname, i18n);
</script>

<LanguageSwitcher
  locales={i18n.locales}
  currentLocale={currentLocale}
  basePath="/docs"
/>
```

## API Reference

### I18nConfig

```typescript
interface I18nConfig {
  /** Default locale code (e.g., 'en') */
  defaultLocale: string;

  /** Supported locales */
  locales: LocaleConfig[];
}

interface LocaleConfig {
  /** Locale code (e.g., 'en', 'es', 'zh-CN') */
  code: string;

  /** Display label (e.g., 'English', 'Español') */
  label: string;

  /** Text direction (default: 'ltr') */
  dir?: 'ltr' | 'rtl';
}
```

### Utilities

#### `detectLocaleFromPath(pathname, config)`

Extract locale from URL path.

```typescript
import { detectLocaleFromPath } from '@goobits/docs-engine/utils';

const locale = detectLocaleFromPath('/es/docs/guide', i18n);
// Returns: 'es'

const defaultLocale = detectLocaleFromPath('/docs/guide', i18n);
// Returns: 'en' (default)
```

#### `getLocalizedPath(pathname, locale, config)`

Convert path to different locale.

```typescript
import { getLocalizedPath } from '@goobits/docs-engine/utils';

const spanishPath = getLocalizedPath('/docs/guide', 'es', i18n);
// Returns: '/es/docs/guide'

const englishPath = getLocalizedPath('/es/docs/guide', 'en', i18n);
// Returns: '/docs/guide' (no prefix for default locale)
```

#### `loadLocalizedContent(path, locale, config)`

Load content with fallback chain.

```typescript
import { loadLocalizedContent } from '@goobits/docs-engine/utils';

// Try es-MX → es → en
const content = await loadLocalizedContent(
  'getting-started.md',
  'es-MX',
  i18n
);
```

### Components

#### LanguageSwitcher

Dropdown for locale selection.

```svelte
<LanguageSwitcher
  locales={i18n.locales}
  currentLocale="en"
  basePath="/docs"
/>
```

**Props:**
- `locales` - Array of available locales
- `currentLocale` - Active locale code
- `basePath` - Documentation base path (e.g., '/docs')

## Routing Setup

### SvelteKit Route Structure

```
src/routes/
  docs/[...path]/+page.svelte              # English (default)
  [locale]/docs/[...path]/+page.svelte     # Other locales
```

### Load Function

```typescript
// src/routes/[locale]/docs/[...path]/+page.ts
import { loadLocalizedContent } from '@goobits/docs-engine/utils';
import { i18n } from '$lib/config';

export async function load({ params }) {
  const { locale, path } = params;

  const content = await loadLocalizedContent(
    path,
    locale,
    i18n
  );

  return {
    content,
    locale,
    availableLocales: i18n.locales
  };
}
```

## RTL Support

Languages with `dir: 'rtl'` automatically apply right-to-left layout:

```typescript
{
  code: 'ar',
  label: 'العربية',
  dir: 'rtl'  // Auto-applies RTL styles
}
```

**Automatic Features:**
- Text alignment reverses
- Reading direction flips
- Component layout mirrors
- Navigation order inverts

## SEO & Metadata

### Hreflang Tags

Automatically generated for all locales:

```html
<link rel="alternate" hreflang="en" href="https://example.com/docs/guide" />
<link rel="alternate" hreflang="es" href="https://example.com/es/docs/guide" />
<link rel="alternate" hreflang="zh" href="https://example.com/zh/docs/guide" />
<link rel="alternate" hreflang="x-default" href="https://example.com/docs/guide" />
```

### Sitemap Generation

Generate separate sitemaps per locale:

```typescript
import { generateI18nSitemap } from '@goobits/docs-engine/server';

const sitemaps = await generateI18nSitemap(navigation, i18n, {
  baseUrl: 'https://example.com'
});

// Returns:
// {
//   en: 'sitemap-en.xml',
//   es: 'sitemap-es.xml',
//   zh: 'sitemap-zh.xml'
// }
```

## Search Integration

### Locale-Specific Indexes

Build separate search indexes per locale:

```typescript
import { buildSearchIndex } from '@goobits/docs-engine/utils';

for (const locale of i18n.locales) {
  const index = await buildSearchIndex({
    locale: locale.code,
    content: await loadDocsForLocale(locale.code)
  });

  await saveSearchIndex(index, locale.code);
}
```

### Search Modal with i18n

```svelte

  const currentLocale = 'es';

<SearchModal
  searchIndex={`/search-index-${currentLocale}.json`}
  placeholder="Buscar documentación..."
  locale={currentLocale}
/>
```

## Fallback Chains

Support regional variations with fallback logic:

```typescript
// User requests: es-MX (Spanish - Mexico)
// Fallback chain: es-MX → es → en

const i18n: I18nConfig = {
  defaultLocale: 'en',
  locales: [
    { code: 'en', label: 'English' },
    { code: 'es', label: 'Español' },      // Fallback for es-*
    { code: 'es-MX', label: 'Español (México)' }
  ]
};
```

**Logic:**
1. Try exact match: `es-MX`
2. Try language code: `es`
3. Fall back to default: `en`

## Translation Workflow

### Recommended Process

1. **Write in Default Locale**
   ```bash
   docs/en/getting-started.md
   ```

2. **Create Locale Directory**
   ```bash
   mkdir -p docs/es
   ```

3. **Copy & Translate**
   ```bash
   cp docs/en/getting-started.md docs/es/getting-started.md
   # Edit docs/es/getting-started.md with translations
   ```

4. **Maintain Structure**
   Keep identical filenames and directory structure across locales.

### Translation Tools (External)

Use external tools for translation management:
- **Crowdin** - Translation management platform
- **Lokalise** - Localization automation
- **Phrase** - Translation workflow
- **POEditor** - Collaborative translation

*Note: docs-engine provides infrastructure, not translation management.*

export const i18n: I18nConfig = {

### Layout with Language Switcher

```svelte
<!-- src/routes/+layout.svelte -->
<script lang="ts">
  import { LanguageSwitcher, DocsLayout } from '@goobits/docs-engine/components';
  import { detectLocaleFromPath } from '@goobits/docs-engine/utils';
  import { page } from '$app/stores';
  import { i18n } from '$lib/config';

  const currentLocale = $derived(detectLocaleFromPath($page.url.pathname, i18n));
  const isRTL = $derived(
    i18n.locales.find(l => l.code === currentLocale)?.dir === 'rtl'
  );
</script>

<div class="layout" dir={isRTL ? 'rtl' : 'ltr'}>
  <header>
    <nav>
      <LanguageSwitcher
        locales={i18n.locales}
        currentLocale={currentLocale}
        basePath="/docs"
      />
    </nav>
  </header>

  <slot />
</div>
```

## Best Practices

### File Organization

**Do:**
```
docs/
  en/
    guide/
      getting-started.md
      configuration.md
  es/
    guide/
      getting-started.md
      configuration.md
```

**Don't:**
```
docs/
  guide/
    getting-started.en.md
    getting-started.es.md
```

### Content Strategy

- Start with default locale (usually English)
- Translate high-traffic pages first
- Keep URL structure identical across locales
- Use locale-specific examples when relevant

### Performance

- Lazy-load locale-specific search indexes
- CDN cache by locale (e.g., `/es/*`)
- Pre-render all locales at build time
- Use locale in cache keys

## Troubleshooting

### Language Switcher Not Showing

**Issue:** Switcher doesn't appear

**Solution:** Verify locale detection:
```typescript
console.log(detectLocaleFromPath($page.url.pathname, i18n));
```

### Wrong Locale Detected

**Issue:** Always shows default locale

**Solution:** Check URL structure matches `[locale]` pattern:
```
✅ /es/docs/guide
❌ /docs/guide?lang=es  (query params not supported)
```

### RTL Layout Not Applied

**Issue:** Arabic content shows LTR

**Solution:** Ensure `dir` attribute on root element:
```svelte
<div dir={isRTL ? 'rtl' : 'ltr'}>
```

## Migration Guide

### From Single Language

1. Create locale directories:
   ```bash
   mkdir docs/en
   mv docs/*.md docs/en/
   ```

2. Update routes to include `[locale]` param

3. Add i18n config with single locale initially:
   ```typescript
   { defaultLocale: 'en', locales: [{ code: 'en', label: 'English' }] }
   ```

4. Add additional locales when ready
